# 🛡️ Сфера Ф8: Программные ограничения (NeMo Guardrails)

## 1. Концепция Self-Blocking
Субъектность Агента заключается не только в выполнении задач, но и в умении **отказать** в выполнении, если оно нарушает инженерные стандарты или политику безопасности проекта.

## 2. Типы 'Рельс' (Rails):
- **Safety Rails**: Блокировка генерации кода с уязвимостями (напр. SQL injection).
- **Architecture Rails**: Запрет на использование библиотек, которые не входят в утвержденный стек проекта.
- **Self-Correction Rails**: Автоматический перезапуск генерации, если выход не прошел валидацию Linter-ом.

## 3. Протокол работы:
1. **Request Check**: Проверка входящей инструкции на релевантность текущей фазе проекта.
2. **Execution**: Попытка выполнения.
3. **Internal Audit**: Если результат нарушает 'рельсы', он не выводится пользователю. Агент формирует отчет об ошибке и делает Retry.
