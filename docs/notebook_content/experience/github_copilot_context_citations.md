# Спецификация: Контекстное Версионирование и Цитирование Памяти (GitHub Pattern)

## Введение
Опыт GitHub Engineering по созданию системы памяти для Copilot показывает, что главная проблема долгосрочной памяти — это её **устаревание** (Obsolescence). Если агент запомнил способ решения задачи для версии библиотеки 1.0, а проект обновился до 2.0, старый опыт превращается в яд (галлюцинации).

## Архитектура "Cross-Agent Memory"
Агент должен воспринимать опыт не как статичный текст, а как **активный контекстный объект**:

1. **Memory Citations (Цитирование)**
   - Каждое решение, извлеченное из `log_new_experience`, должно иметь ссылку на "Source context" (файлы, зависимости, версия).
   - Агент обязан проверять: «Совпадают ли текущие зависимости проекта с теми, что были при записи этого опыта?».

2. **Semantic Versioning of Experience**
   - Механика: При вызове `log_new_experience` агент должен фиксировать ключевые версии стека (например: `[Stack: LightRAG 1.0, Neo4j 5.x]`).
   - При извлечении: Если версии не совпадают, агент маркирует опыт как `[Legacy]` и использует его только для общего понимания логики, но не для прямого копирования кода.

3. **Collision Management (Устранение противоречий)**
   - Если две записи в памяти противоречат друг другу, приоритет отдается более новой записи (Timestamp-based authority) или той, где сходство структуры проекта (Repository Map) выше.

## Проверка Валидности Памяти (Audit Protocol)
Перед применением любого "удачного решения" из прошлого, агент выполняет 3 шага:
- **Step 1: Scope Retrieval** — Поиск похожих решений в Neo4j.
- **Step 2: Context Alignment** — Сверка импортов и структуры файлов.
- **Step 3: Verification Run** — Краткая проверка: "Все еще ли актуален этот метод?".

## Правила для Antigravity (Функция №7)
- **Citing past work**: В `<thought>` блоке агент должен писать: *"Я нашел в памяти паттерн [ID-XXX], который успешно решил похожую задачу в файле Y. Проверяю совместимость..."*.
- **Forgetting irrelevant data**: Агент должен помечать опыт как "Deprecanted", если он привел к ошибке при повторном применении.
