# 🔖 Сфера Ф10: XML-инкапсуляция и чистота контекста (Anthropic Style)

## 1. Зачем нужны теги?
Для LLM огромный блок текста — это единое полотно. Использование XML-тегов создает «логические границы», которые модель считывает как границы модулей.

## 2. Обязательные теги для Агента:
- `<rules>`: Здесь фиксируются запреты.
- `<source_code>`: Только чистый код без лишних объяснений.
- `<thought>`: Скрытый слой рассуждений.
- `<action_json>`: Машиночитаемая команда для выполнения.

## 3. Преимущества Traceability:
Когда ответ Агента обернут в теги, разработчик (или другой Агент-Эвалюатор) может мгновенно распарсить его и проверить только блок `<thought>` на предмет логических ошибок, не тратя токены на весь текст.
