# Критический разбор ошибок синхронизации NotebookLM (V3)

### Детальный разбор провала Deep Sync (RCA)

**Проблема**: Использование `notebook_query` для итеративного дампа ("Step 1, Step 2...") привело к пустым ответам.
**Решение**:
1. Переход на JIT (Just-in-Time) извлечение.
2. Обязательная проверка `if not answer: raise Error` перед записью.
3. Использование `source_get_content` (если доступно) или прямого Browser-парсинга для дословного копирования.
**Статус**: Анти-паттерн зафиксирован. Запрещено использовать старый метод `deep_sync_notebooklm.py`.
