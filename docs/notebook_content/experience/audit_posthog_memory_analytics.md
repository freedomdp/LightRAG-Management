# 📈 Сфера Ф10: Аналитика контекста и вытеснение памяти (PostHog)

## 1. Метрики эффективности памяти
Агент должен не просто «помнить», а осознавать, сколько ресурсов он потребляет.
- **Cost Analytics**: Стоимость одного шага рассуждения.
- **Context Utilization**: Какой процент из 128k окна был действительно релевантен текущему фиксу.

## 2. Стратегии Pruning (Очистки)
Когда контекст достигает 80% лимита, Агент инициирует `Context Eviction`:
1. Удаление всех промежуточных шагов `grep` и `ls`.
2. Суммаризация принятых решений.
3. Сохранение только активного `task.md` и открытых файлов.

## 3. Версионирование состояний
Каждое важное архитектурное решение помечается в PostHog как событие (Event). Это позволяет увидеть «воронку» (Funnel) ошибок и понять, на какой стадии проекта Агент чаще всего начинает галлюцинировать.
