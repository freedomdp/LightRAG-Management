# Провал глубокой синхронизации NotebookLM (Empty Results)

### Провальный опыт: Глубокая синхронизация из NotebookLM через notebook_query

**Проблема**: Попытка получить дословный (verbatim) контент через итеративные запросы `notebook_query` ("Extract Part 1...") привела к получению пустых ответов от API.
- Инструмент возвращал `{"status":"success", "answer": ""}`.
- Скрипт `deep_sync_notebooklm.py` записывал эти пустые ответы в файлы, что привело к засорению базы данных LightRAG "пустыми" документами.

**Последствия**:
- Файловая система заполнена `part_X.md` файлами без полезного контента.
- База данных (Neo4j/Qdrant) проиндексировала эти пустые сущности, что может привести к деградации качества RAG-ответов.

**Рекомендации**:
1. Никогда не доверять статусу `success` от NotebookLM MCP без проверки непустого поля `answer`.
2. Сообщение `Extract the full technical instructions...` может быть слишком сложным для инструмента `notebook_query`, если он не может фокусироваться на конкретном источнике так же эффективно, как на всем блокноте.
3. Перед массовой загрузкой проверять объем каждого полученного чанка.

**Статус**: Опыт зафиксирован для предотвращения повторения. Идет разработка плана очистки.
