# Исправление логики прогресса индексации: переход на физический учет файлов (A/B)

### Проблема
Логика дашборда (V3.0-V3.2) некорректно отображала прогресс индексации документов. В поле "Реальная обработка (Neo4j)" выводилось [5/5] (100%), основываясь на данных `ingest_status.json`, который фиксировал только факт *закачки* файлов в API, но не завершение их индексации в базе данных. Это вводило пользователя в заблуждение, так как фактически в этот момент еще шли процессы чанкинга.

### Решение
1.  **Физический расчет**: Переписал метод `build_dashboard` в `scripts/dashboard_engine.py`. Теперь "B" (Всего) рассчитывается через `len(os.listdir('docs/notebook_content'))`.
2.  **Замкнутый цикл подтверждения**: Добавил в `dashboard_state.json` список `indexed_files`. Файл попадает в этот список только в момент появления записи `Chunk X of X` в логах Docker API (где X - финальный чанк).
3.  **Синхронизация Правил**: Обновил `docs/DASHBOARD_V3_RULES.md`, удалив все упоминания JSON как источника для `total`. Установлен стандарт: `B` - это папка, `A` - это подтвержденная база.

### Результат
Дашборд теперь честно показывает `[0/5]`, если файлы в папке есть, но база еще не подтвердила их завершение. Это гарантирует 100% прозрачность процесса для пользователя.
