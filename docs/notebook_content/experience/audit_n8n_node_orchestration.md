# 🧩 Сфера Ф8: Визуальная оркестрация и нодовая логика (n8n)

## 1. Задачи как узлы
В парадигме n8n любая задача — это независимый узел (Node) с четким входом и выходом. Это предотвращает «побочные эффекты» (Side Effects) при работе Агента.

### Ключевые паттерны:
- **Webhooks**: Внешние триггеры, запускающие цепочку рассуждений.
- **HTTP Request**: Основной инструмент взаимодействия Агента с другими сервисами. Каждый вызов должен иметь таймаут и обработку ошибок (Retry logic).
- **IF/Switch Nodes**: Агент обязан проверять условия выполнения задачи на каждом этапе, не допуская автоматического перехода к следующему шагу при сбое.

## 2. Изоляция данных в нодах
Агент учится тому, что данные между узлами передаются в виде чистых JSON-объектов. Нельзя обращаться к глобальным переменным проекта, если они не переданы в текущий контекст узла.

## 3. Масштабируемость (Scale)
Использование n8n позволяет Агенту разделять огромные проекты на десятки маленьких воркфлоу (Sub-workflows), которые вызываются рекурсивно. Это решает проблему переполнения контекстного окна LLM.
