# Спецификация: Агрегация и Суммаризация Опыта (Elastic Pattern)

## Введение
Инженеры Elastic подчеркивают критическую важность превращения «сырых» логов активности в структурированный контекст. Без суммаризации агент утонет в бесконечном потоке трейсбеков и сообщений.

## Методология Агрегации Памяти

1. **Short-term to Long-term Consolidation**:
   - Каждое действие логгируется (Short-term).
   - Раз в сессию или по триггеру успеха происходит **схлопывание (Collapse)** логов в одну запись опыта в Neo4j (Long-term).

2. **Context Collision Management (Управление коллизиями)**:
   - Когда два фрагмента памяти относятся к одной задаче, но предлагают разные методы, агент должен уметь сравнивать их «эффективность» (количество шагов до успеха).
   - *Для LightRAG*: Мы используем вес связей в графе для оценки релевантности.

3. **Semantic Abstraction**:
   - Агент должен уметь абстрагироваться от конкретных имен переменных к паттерну решения. 
   - *Пример*: Из опыта «Фикс race condition в dashboard_engine.py» агент должен уметь извлечь паттерн «Atomic file replacement through temp files» для применения в любом другом модуле.

## Практическая реализация
- **Memory Pruning**: Удаление дублирующих или шумовых записей.
- **Span-based Tracing**: Группировка опыта по «Пролетам» (Spans) — смысловым этапам задачи.

## Правила для Antigravity
Агент не имеет права записывать в опыт "мусор". Каждая запись в `log_new_experience` должна пройти внутренний тест на **Summarization Value**: *"Будет ли это полезно мне при решении ДРУГОЙ задачи через месяц?"*.
