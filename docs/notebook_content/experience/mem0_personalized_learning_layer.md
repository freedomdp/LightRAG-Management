# Спецификация: Самосовершенствующийся Слой Памяти (Mem0 Pattern)

## Обзор
Mem0 (Memory Layer) представляет подход, при котором память агента — это не просто хранилище текста, а динамический слой «фактов», который постоянно обновляется и самосовершенствуется.

## Ключевые принципы Mem0 для LightRAG

### 1. Persistent Fact Extraction (Извлечение фактов)
Вместо того чтобы хранить: *"Пользователь попросил меня исправить баг в dashboard_engine.py на строке 286 и я заменил rename на replace"*, агент должен сохранить дистиллированный факт:
- **Fact**: `[dashboard_engine.py] replace is more reliable than rename for atomic updates`.
- **Level**: System-Standard.

### 2. User Preference Learning (Персонализация)
Агент должен запоминать предпочтительный стиль работы разработчика:
- *"Пользователь предпочитает XML-теги для рассуждений"*.
- *"Пользователь требует использовать -y для npx"*.
Эти факты становятся «призрачными инструкциями», которые автоматически добавляются в контекст при извлечении опыта.

### 3. Forget & Update (Забвение и Обновление)
Память должна быть способна перезаписывать себя. 
- Если новое «удачное решение» противоречит старому, старое помечается как `OUTDATED`.
- Это решает проблему конфликтов в графе знаний.

## Правила для Antigravity
- **Continuous refinement**: Каждый вызов `log_new_experience` — это возможность уточнить уже существующий опыт.
- **Context injection**: Перед запуском любой команды `run_command` агент должен "вспомнить" предпочтения пользователя по безопасности и параметрам.
