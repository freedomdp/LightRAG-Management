# 🛡️ Сфера Ф8: Guardrails и Жесткая типизация форматов

## 1. Проблема «Галлюцинаций формата»
Галлюцинации часто возникают не в логике, а в представлении данных (лишние скобки в JSON, неверный тип поля). Для обеспечения субъектности Агент должен использовать аппарат «Ограниченного декодирования» (Constrained Decoding).

## 2. Structured Outputs (OpenAI/Standard JSON Schema)
Агент обязан уметь отдавать результат по строгой схеме. В сложных задачах (напр. генерация архитектурного дерева) используется `strict: true` (имитация).

### Правила работы со схемами:
- **Zero Additional Properties**: Запрещено добавлять поля, не описанные в ТЗ.
- **Required Fields**: Все критические поля (Path, ID, Impact) должны быть заполнены.
- **Validation**: Прежде чем отправить JSON пользователю, Агент делает «внутреннюю десериализацию», чтобы убедиться в его корректности.

## 3. Концепция «Рельсов» (NeMo Guardrails)
Это набор физических блокировщиков (programmable rails), которые контролируют вход и выход.
- **Topical Rails**: Если Агент начинает рассуждать о темах, не относящихся к архитектуре проекта, генерация останавливается.
- **Dialog Rails**: Запрет на «импровизацию» ответов без опоры на базу знаний LightRAG.
- **Input Rails**: Фильтрация входящего ТЗ на предмет противоречий (напр. «использовать React и Flutter одновременно в одном модуле»).

## 4. XML-инкапсуляция контекста (Anthropic Style)
Для изоляции кусков ТЗ и предотвращения каши в контексте, Агент использует теги:
- `<rules>`: Жесткие правила и ограничения.
- `<context>`: Ссылки на существующий код.
- `<constraints>`: Что НЕЛЬЗЯ делать.
- `<output_schema>`: Ожидаемый формат.

**Benefit**: Это позволяет LLM «не путать» переменные из разных модулей, обрабатывая каждый тег как изолированный логический блок.

## 5. Structured Decisions (Субъектность Архитектора)
Архитектурное решение не должно быть «вольным текстом». Оно должно содержать:
1. **Decision (JSON)**: Стандартизированное описание.
2. **Rationale (MD)**: Человеческое объяснение.
3. **Trace (XML)**: Путь рассуждения для Observability.
