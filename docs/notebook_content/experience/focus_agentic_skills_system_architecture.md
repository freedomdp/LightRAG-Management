# 🛠️ Сфера Ф5: Архитектура Системы Скиллов (Agentic Skills)

## 1. Определение и Назначение
Скиллы (Skills) — это модульные расширения возможностей Агента, организованные по принципу «инструкция + ресурсы + скрипты». В проекте LightRAG они располагаются в директории `.agents/skills`.

**Mandatory Protocol**: При получении задачи Агент первым делом обязан проверить наличие релевантного скилла через инструменты поиска или `list_dir`. Если скилл найден, его файл `SKILL.md` становится высшим приоритетом управления (Primary Instruction).

## 2. Структура Скилла
Каждый скилл представляет собой папку со следующим содержимым:
- **SKILL.md** (Обязательно): Содержит YAML-метаданные (название, описание) и детальные инструкции.
- **scripts/**: Вспомогательные скрипты для автоматизации задач.
- **resources/**: Шаблоны, активы или дополнительные данные.
- **examples/**: Референсные реализации.

## 3. Механизм активации (Skill Lifecycle)
1. **Discovery**: Агент сканирует базу знаний или ФС на наличие скилла (например, `task-manager`).
2. **Context Loading**: Файл `SKILL.md` считывается целиком.
3. **Execution**: Агент интегрирует инструкции скилла в текущий план действий.
4. **Log**: Результаты использования скилла могут быть зафиксированы через `log_new_experience`.

## 4. Текущий реестр (Snapshot)
- `agent-memory-mcp`: Система гибридной памяти.
- `filesystem-context`: Динамическое управление контекстом через файлы.
- `simple-fetch`: Базовое извлечение данных.
- `task-manager`: Управление задачами через GitHub Issues.

## 5. Практика применения (Best Practices)
- **Don't Overload**: Скилл должен решать одну конкретную задачу (SRP - Single Responsibility Principle).
- **Update Metadata**: При изменении возможностей скилла его описание в SKILL.md должно быть обновлено.
- **Recursive Skills**: Скиллы могут ссылаться друг на друга для выполнения сложных операций.
