# Спецификация: 5 Уровней Агентной Памяти и Самообучение (Agno Pattern)

## Обзор
Настоящий документ описывает архитектурный фреймворк Agno для построения надежных ИИ-агентов, с особым фокусом на механизмы «GPU-Poor Learning» (самообучение без дообучения весов модели).

## Уровни Агентного ПО (Эволюция)
Для достижения истинной автономности агент должен пройти путь от статического исполнителя до самообучающейся системы:

1. **Level 1: Stateless Execution**
   - Самый простой уровень. Каждое сообщение изолировано. Нет памяти.
   - *Проблема*: Агент совершает одни и те же ошибки в каждом цикле.

2. **Level 2: Basic Working Memory**
   - Использование истории чата (Short-term context).
   - *Проблема*: Линейное накопление контекста ведет к деградации внимания («Lost in the Middle»).

3. **Level 3: Long-term Contextual Memory (RAG)**
   - Извлечение данных из внешних баз под задачу.
   - *Статус LightRAG*: Мы находимся здесь. Агент умеет читать документацию.

4. **Level 4: Continuous Experience Learning (GPU-Poor)**
   - **Критическая зона для этого мануала**. Агент использует инструмент `log_new_experience` для фиксации паттернов.
   - *Механика*: Фиксация "успехов" и "провалов" каскадно обогащает базу знаний. Агент начинает "узнавать" архитектурные паттерны текущего проекта.

5. **Level 5: Recursive Task Optimization**
   - Агент сам перестраивает свои промпты и инструменты на основе накопленного опыта.

## Концепция "GPU-Poor Continuous Learning"
В условиях, когда веса модели заморожены, обучение происходит на уровне **мета-контекста**:

- **Semantic Memory Injection**: Перед началом задачи агент делает поиск по тегу `experience`, чтобы найти аналогичные кейсы из прошлого.
- **Style Consistency**: Сохранение предпочтений пользователя (например, «всегда используй os.replace вместо rename») становится частью «ДНК» агента.
- **Fail-Fast Recovery**: Если в памяти есть запись о провальной попытке решить задачу X способом Y, агент обязан выбрать путь Z.

## Практические правила для LightRAG (Функция №7)
1. **Zero Repeated Errors**: Перед написанием кода, решающего системную проблему, агент делает RAG-запрос: `[Search: "issue solved" OR "failed attempt"]`.
2. **Context Compression**: Вместо сохранения всего лога переписки в опыт, агент должен выделять **дистиллированное правило** (Distilled Logic).
3. **Triggered Archiving**: Любое сообщение пользователя вида "Это сработало" является императивным сигналом к записи нового паттерна.
