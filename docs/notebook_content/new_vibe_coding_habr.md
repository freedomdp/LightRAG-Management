TITLE: Как работать с Claude Code, Antigravity и Codex в 2026: база вайбкодинга / Хабр
CONTENT: 
Вайбкодинг — это не "сделай мне приложение", а управляемый процесс.
Базовый пайплайн: спецификация → план → маленькие итерации → тесты/ревью → фиксация изменений (git).

1. Спецификация (spec.md): Опишите идею, попросите модель задавать вопросы. Согласуйте требования и крайние случаи.
2. План проекта: Разбейте реализацию на логичные этапы (используйте thinking-модели).
3. Маленькие шаги (тикеты): Не делайте фундаментальных запросов, идите поэтапно.
4. Контекст и MCP: ИИ должен видеть файлы, архитектурные запреты и актуальные доки через Model Context Protocol.
5. Выбор моделей: Разные модели для кодинга, рефакторинга и анализа логов.
6. Агентные CLI (Claude Code, Codex, Antigravity): Используйте только ПОСЛЕ спеки и плана. Агент сам читает файлы и запускает тесты.
7. Тесты: Контроль качества на каждом шаге.
8. Коммиты: Ваши точки сохранения (save points). Делайте коммит после каждого успешного шага.
9. Правила проекта (CLAUDE.md / GEMINI.md): Линтеры, стиль, запреты.

Вайбкодинг — это контролируемый цикл, ускоряющий механику, а не увеличивающий хаос.
