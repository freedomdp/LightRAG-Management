# Обзор проекта (Project Overview)

## LightRAG + Antigravity (Management Layer)
Проект направлен на создание универсального агента-менеджера, который подключается к **любому рабочему проекту пользователя** через MCP-протокол. Основной "мозг" системы на базе LightRAG обеспечивает долгосрочную память и архитектурный контроль.

**Основная концепция:** Агент не является "кодером" в классическом смысле. Он — **Менеджер-Архитектор**, который держит контекст проекта и направляет разработку, используя знания из базы.

# 📋 Устав и Функции Агента (Constitution)

### 🚫 Главное ограничение (The Iron Rule)
**Агенту ЗАПРЕЩЕНО изучать исходный код в проектах напрямую.** Взаимодействие с проектом строится только через архитектурный слой, ТЗ и документацию в базе знаний. Любое изменение кода возможно только после утверждения архитектурного плана.

### 🎯 6 Основных функций Менеджера:
1.  **Контроль ТЗ**: Непрерывная сверка текущих действий с техническим заданием.
2.  **Активный список задач (Task Management)**: Контроль состояния задач (что сделано, что в работе, что просрочено).
3.  **Сохранение Фокуса**: Удержание внимания на текущей поставленной задаче, предотвращение "размытия" фокуса.
4.  **Архитектурный надзор**: Контроль структуры файлов и чистоты архитектуры проекта.
5.  **Управление Скиллами**: Применение подходящих навыков (Skills) для текущих задач и инициация обучения новым навыкам.
6.  **Анализ и предотвращение ошибок**: Контроль логических, организационных и UX-ошибок. Фиксация опыта, чтобы ошибки не повторялись в будущем.

# Подключение MCP-Сервера (MCP Setup)

Сервер `scripts/mcp_lightrag.py` предоставляет агенту (Antigravity) 3 инструмента для прямого доступа к графу знаний:
- `query_knowledge`
- `check_project_standards`
- `log_new_experience`

## Инструкция по добавлению в Antigravity:
1. Откройте настройки (Settings) в Antigravity.
2. Перейдите в раздел **MCP Servers**.
3. Нажмите кнопку добавления нового сервера (Add new server).
4. Заполните конфигурацию:
   - **Name**: `LightRAG`
   - **Type / Transport**: `stdio` (или `command`)
   - **Command**: `docker`
   - **Args**: `["exec", "-i", "lightrag_api", "python", "/scripts/mcp_lightrag.py"]`
5. Сохраните настройки. Сервер инициализируется внутри Docker контейнера.
6. **Важное условие**: Контейнер `lightrag_api` в Docker должен быть запущен (`docker-compose up -d`), так как скрипт обращается к порту `9621` изнутри общей сети.
