# Полное руководство по интеграции BrowserAct с n8n

Данное руководство описывает современный и наиболее стабильный способ извлечения данных из веб-сайтов с помощью связки BrowserAct и n8n.

## 1. Архитектурный обзор
Традиционные методы извлечения данных (через прямые HTTP-запросы к `/v1/extract`) считаются устаревшими. BrowserAct теперь функционирует как **Workflow Engine**. Каждое извлечение — это выполнение воркфлоу, что позволяет гибко настраивать обход капчи, задержки и специфическую логику парсинга.

## 2. Предварительная настройка сервера (Docker)
Для установки официальной ноды BrowserAct в n8n необходимо разрешить использование пакетов сообщества.

1. Откройте ваш `docker-compose.yml`.
2. Добавьте следующую переменную в раздел `environment` для сервиса n8n:
   ```yaml
   environment:
     - N8N_COMMUNITY_PACKAGES_ENABLED=true
   ```
3. Перезапустите контейнеры:
   ```bash
   docker compose up -d
   ```

## 3. Установка ноды в интерфейсе n8n
1. Перейдите в **Settings** (Настройки) в левом нижнем углу n8n.
2. Откройте раздел **Community Nodes** (Ноды сообщества).
3. Нажмите кнопку **Install a community node**.
4. В поле `npm Package Name` введите: `n8n-nodes-browseract`.
5. Нажмите **Install**. После завершения установки в списке узлов n8n появится "BrowserAct".

## 4. Настройка авторизации (Credentials)
1. Перейдите в консоль BrowserAct: [integrations & API](https://www.browseract.com/dashboard/integrations).
2. Создайте новый API Key.
3. В n8n создайте новые **Credentials** (Учетные данные).
4. Найдите тип `BrowserAct API` и вставьте ваш ключ.

## 5. Работа с узлом BrowserAct
Вместо использования `HTTP Request`, добавьте на холст узел **BrowserAct**.

### Основные настройки:
- **Resource**: `Workflow`
- **Operation**: `Run a Workflow`
- **Workflow Source**: `Template Marketplace` — это позволяет использовать готовые, оптимизированные шаблоны.
- **Workflow**: Выберите шаблон `Smart Extract` (наилучший выбор для универсального парсинга цен и характеристик).

### Входные параметры (Input Parameters):
- **URL**: Ссылка на целевую страницу товара.
- **User Prompt**: Инструкция для ИИ (например: *"Extract price as number and inventory status as boolean"*).
- **Result Wait**: Рекомендуется значение `Wait For Result`, чтобы n8n дождался окончания скрапинга перед переходом к следующей ноде.

## 6. Обработка результатов
Нода возвращает структурированный JSON. Пример маппинга данных в n8n:
- Цена: `{{ $json.data.price }}`
- Наличие: `{{ $json.data.available }}`

## 7. Решение проблем (Troubleshooting)
- **Ошибка 404**: Возникает при попытке обращения к старым эндпоинтам. Переход на официальную ноду полностью решает эту проблему.
- **Таймауты**: Для сложных страниц рекомендуется увеличивать таймаут в настройках ноды (раздел Settings в настройках узла).
