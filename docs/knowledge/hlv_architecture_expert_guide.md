# Архитектура HLV (Human-LLM-Validation): Глубокий стандарт разработки

Архитектура HLV — это система **«Контрактного программирования для ИИ»**, разработанная для устранения недетерминированности (рандома) LLM-агентов. Она превращает процесс генерации кода в строгую инженерную дисциплину, где каждый шаг подтверждается автоматизированной валидацией.

---

## 1. Трехслойная структура (The Three Layers)

### 🔵 Слой Человека (Human Layer — "The What")
Здесь определяются требования и правила игры. Это «Источник истины» (Source of Truth), который не меняется в процессе генерации кода.
- **Glossary (`glossary.yaml`)**: Бизнес-объекты и типы данных, описанные через JSON Schema.
- **Contracts (`contracts/*.yaml`)**: Формальные спецификации каждой функции или операции.
- **Constraints (`constraints/`)**: Ограничения (безопасность, производительность, архитектура).
- **Map (`map.yaml`)**: Индекс всех файлов и их ролей в проекте, чтобы LLM знала, где искать контекст.

### 🟢 Слой LLM (LLM Layer — "The How")
Здесь происходит реализация логики. Агент использует артефакты из Human Layer как техническое задание и документацию.
- **Implementation**: Код, написанный на любом языке (Rust, Python, JS).
- **Tracing**: Обязательная расстановка **маркеров** в коде для связи реализации с контрактами.

### 🔴 Слой Валидации (Validation Layer — "The Guard")
Слой автоматического контроля, реализованный в виде Rust-бинарника (CLI `hlv`).
- **Checkers**: Скрипты и правила, проверяющие код на соответствие контрактам.
- **Gates**: Гейты (ворота), которые блокируют завершение задачи, если хотя бы одно условие не выполнено.

## 2. Техническая реализация (Deep Specs)

### 🏗️ Система Маркеров (Traceability Markers)
Маркеры — это специальные комментарии, которые связывают код с пунктами контрактов.
- **Синтаксис**: `// @hlv <CONTRACT_ID>` или `# @hlv <CONTRACT_ID>`.
- **Пример**: Если контракт `USER_AUTH` требует обработки ошибки `INVALID_PASSWORD`, в коде **должен** быть маркер:
  ```javascript
  if (password !== hashed) {
      // @hlv INVALID_PASSWORD
      throw new AuthError("Wrong password");
  }
  ```
- **Валидация**: `hlv check` парсит код и выдает ошибку, если в контракте описана логика, для которой в коде нет соответствующего маркера.

### 🧱 Контракты (Contracts)
Каждый контракт в `human/contracts/` должен содержать:
- `inputs_schema`: Схема входящих данных.
- `outputs_schema`: Схема результата.
- `errors`: Список возможных ошибок с полем `when` (условие возникновения).
- `invariants`: Условия, которые никогда не должны нарушаться.

### 🚪 Гейты (Gates Policy)
Описываются в `validation/gates-policy.yaml`. Гейт — это команда, которая ДОЛЖНА вернуть успех.
```yaml
gates:
  - id: contract_coverage
    description: "Все ли условия контрактов помечены маркерами в коде?"
    command: "hlv check --markers"
  - id: security_audit
    description: "Проверка на наличие критических уязвимостей в помеченных @hlv:sec местах."
    command: "hlv scan-security"
```

## 3. Сравнение: AI Factory vs HLV

| Характеристика | AI Factory | HLV Architecture |
| :--- | :--- | :--- |
| **Цель** | Массовая генерация (Quantity) | Инженерное качество (Quality) |
| **Сложность** | Шаблонные проекты (Landing, CRUD) | Сложные системы, бизнес-логика |
| **Роль LLM** | Исполнитель «под ключ» | Архитектор-кодер в жестком контуре |
| **Риск** | Средний (галлюцинации допустимы) | Нулевой (валидатор блокирует ошибки) |

## 🚀 Workflow (Алгоритм работы)

1.  **Инициализация (`hlv init`)**: Создание структуры и базовых конфигов.
2.  **Артефакты**: Написание контрактов и глоссария в `human/`. Это делается человеком или агентом-архитектором.
3.  **Индексация**: Обновление `map.yaml` для LLM.
4.  **Кодинг**: Генерация реализации с обязательной расстановкой маркеров.
5.  **Верификация (`hlv check`)**: Запуск валидатора. Если гейты не пройдены — возврат в шаг 4 с фидбеком от валидатора.

---

> [!IMPORTANT]
> Для **Vibe Coding** архитектура HLV является «спасательным кругом»: она позволяет «вайбить» быстро, но при этом иметь автоматического «зануду-проверяющего», который не даст сломать логику или забыть про безопасность.
