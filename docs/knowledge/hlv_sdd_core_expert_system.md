# Antigravity AI: Unified Core Expert System (HLV + SDD + Business ROI)

Этот документ является высшим техническим стандартом (Tier-1) для проекта Antigravity. Он объединяет методологии разработки (SDD/HLV) с фундаментальными бизнес-алгоритмами автономных систем.

---

## 🏗️ 1. Архитектурный цикл (SDD + HLV)

Разработка ведется строго по циклу **Spec-Driven Development**, где код — это производный артефакт спецификации:
1.  **Constitution**: Мастер-стандарт проекта (project.md).
2.  **Specification**: Описание логики ("Что" и "Почему").
3.  **Plan**: Техническая декомпозиция (архитектура, API, данные).
4.  **Human Validation (HLV)**: Обязательная пауза для одобрения человеком ПЕРЕД реализацией.
5.  **Task Traceability**: Выполнение через атомарные задачи в `task.md`.
6.  **Drift Guard**: Запрещено изменять код без предварительного обновления спецификации и плана.

---

## 🧠 2. Анатомия Агента (Expert Stack)

На основе стандартов Lilian Weng и Huyen Chip, агент Antigravity состоит из:
- **Planning (Brain)**:
    - **CoT/ToT**: Пошаговое рассуждение и дерево мыслей.
    - **ReAct**: Замкнутый цикл "Рассуждение + Действие".
    - **Self-Reflection (Reflexion)**: Способность агента критиковать и исправлять свои прошлые шаги.
- **Memory (Dual-Layer Resilience)**:
    - **Short-term**: Эффективное использование контекстного окна (учет Lost-in-the-Middle).
    - **Long-term**: Vector DB с использованием MIPS-алгоритмов (HNSW, ScaNN) для быстрого поиска по базе знаний.
- **Control Flows**: Вместо одного промпта — программные паттерны (Sequential, Parallel, For-Loop).

---

## 📈 3. Бизнес-логика и ROI (Strategic Alignment)

Согласно критериям HBR и Twilio/Segment, проект фокусируется на **Value-at-Stake**:
- **ROI Matrix**: Автоматизация только транзакционных задач с четким критерием успеха.
- **Identity Resolution**:
    - **Deterministic Matching**: 100% точность склейки профилей по уникальным ID (Email, Phone).
    - **Identity Graph**: Единый реестр идентификаторов (Cookies, UserID, DeviceID) для омниканальности.
    - **4-Step IR Algorithm**: Collect -> Match -> Merge -> Activate.

---

## 🚨 4. Эскалация и HITL (Handoff Protocol)

Для предотвращения деградации клиентского опыта внедряется **Anti-Dump Protocol**:
- **Triggers**: Эскалация по Трешхолду уверенности (Confidience < 0.7), агрессии клиента (Sentiment) или выходу за Scope.
- **Structured Handover**: Передача оператору не лога чата, а **Structured State**:
    - Global Intent (зачем пришел).
    - Local Summary (что обсудили).
    - Action Log (что уже сделано в API).
- **Graceful Handover**: 30-секундный период параллельной активности сессии при передаче от бота к человеку.

## 🛡️ 5. Гвардейские стандарты (Hard Constraints)

- **Zero-Compression Policy**: Техническая документация вносится в базу знаний без упрощений, сохраняя алгоритмы и математические модели.
- **Fortress Protocol**: Обязательный бэкап данных (Git + DB Backups) перед любыми операциями по изменению ядра.
- **Audit-First**: Каждое техническое решение должно иметь ссылку на проверенный источник (Expert Reference).
