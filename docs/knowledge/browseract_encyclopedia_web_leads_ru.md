# Энциклопедия BrowserAct: Исследование веба и обогащение лидов

Этот документ посвящен возможностям вертикального поиска, анализа веб-сайтов и скрапинга контактов с помощью BrowserAct, что необходимо для **Фазы обогащения (Enrichment Phase)** стратегии M.O.N.E.Y.

## 1. Основные модули
- **web-search-scraper-api-skill**: Скрапинг результатов поиска Google/Bing с высокой точностью. Обходит региональные ограничения и антибот-системы.
- **web-research-assistant**: Динамический агент, способный посещать несколько URL-адресов для ответа на сложные исследовательские вопросы (например, «Найди структуру цен этих 5 конкурентов»).
- **business-contact-social-links-skill**: Мощный инструмент обогащения. На основе URL сайта находит LinkedIn, Twitter, Instagram, Facebook и профессиональные адреса электронной почты.

## 2. Рабочий процесс обогащения (Workflow)
Для построения высококачественной базы лидов следуйте этой последовательности:
1. **Обнаружение (Discovery)**: Используйте навыки Maps/Search для получения URL-адресов сайтов.
2. **Извлечение контактов**: Передайте URL в навык `business-contact-social-links-skill`.
3. **Внутреннее исследование**: Используйте `web-research-assistant` для поиска конкретных «болевых точек», упомянутых на страницах клиента «О нас» или «Кейсы».

## 3. Ключевые технические особенности
### A. Возможности вертикального поиска
- Фильтрация результатов поиска по времени (последние 24 часа, последняя неделя).
- Извлечение данных только с определенных доменов (например, поиск компании только в LinkedIn).

### B. Интеллектуальный поиск контактов
- BrowserAct не просто ищет ссылки `mailto:`.
- Система анализирует лендинги, футеры и страницы контактов, используя Headless-браузинг для поиска скрытых социальных ссылок и контактных форм.

### C. Автоматизированный просмотр (Visit Node)
- Система идеально обрабатывает SPA (Single Page Applications) и сложный JavaScript-рендеринг.
- Узлы «Visit» поддерживают ожидание конкретных элементов, чтобы гарантировать загрузку данных перед скрапингом.
