# Манифест Осознанного Вайб-кодинга (AI Factory 1.8 Standard)

Этот документ суммирует ключевые подходы к разработке программного обеспечения с помощью ИИ-агентов, представленные в релизе **AI Factory 1.8**. Главная цель — переход от хаотичного написания кода («фикс на фикс») к структурированной, поддерживаемой разработке.

## 1. Осознанный vs Хаотичный Вайб-кодинг

| Характеристика | Хаотичный (Chaos Vibe) | Осознанный (Conscious Vibe) |
| :--- | :--- | :--- |
| **Точка входа** | Желание («Хочу кнопку») | Спецификация/Контракт (HLV) |
| **Процесс** | Бесконечный цикл правок | План -> Реализация -> Валидация |
| **Контекст** | Засорен логами и ошибками | Чистая архитектура, строгое управление |
| **Результат** | «Черный ящик», который страшно трогать | Прозрачный, модульный код |

## 2. Spec-Driven Approach (Разработка через спецификации)
Главная проблема ИИ — галлюцинации и потеря контекста. Решение — **Spec-driven first**.
- Перед любой задачей создается **Plan/Spec**.
- Агент не имеет права писать код, пока не подтвержден план.
- В релизе 1.8 это закреплено через обязательный шаг `fixplan.md`.

## 3. Механика проксирования Claude
Опровергаем миф о «засорении» системного промпта:
- **Мета-данные**: В системный промпт Claude (через Claude Code или AI Factory) передаются только *описания* доступных скиллов (schema), а не их код.
- **JIT (Just-In-Time) Loading**: Код скилла или контекст файла подгружается только в момент вызов (Tool Call).
- Это позволяет использовать **14+ специализированных агентов** одновременно без роста потребления токенов в базовом промпте.

## 4. Ключевые нововведения AI Factory 1.8
Релиз 1.8 превращает фабрику в полноценную ОС для разработки:
- **Git Worktree (`feature --parallel`)**: Позволяет запускать разных агентов в изолированных директориях одного репозитория. Больше никаких конфликтов при параллельной работе над UI и Backend.
- **Utility-скиллы**:
    - `Dockerize`: Автоматическая контейнеризация любого проекта.
    - `CI/Build`: Автоматическое создание пайплайнов для GitHub Actions.
    - `Docs`: Авто-генерация README и технической документации в реальном времени.
- **Support for 14+ Agents**: Полная поддержка Cursor, Windsurf, Roo Code, Claude Code и других популярных инструментов в едином воркфлоу.

## 5. Философия качества: «Долой фиксы — строим сразу»
Главный инсайт выпуска: поддержка кода, написанного «быстрым вайбом», стоит в 10 раз дороже самой разработки.
- Обязательная валидация (Layer 3 в HLV).
- Каждый «гейт» должен быть пройден реально (тесты, линтеры, ручная проверка).
- Если агент зациклился на ошибке — **вайн (wipe)** контекста и новый план, а не попытка «дожать» плохой код.

---
**Золотой стандарт**: Архитектура проекта должна быть видна ИИ как на ладони, а не прятаться за сотнями строк неструктурированного кода.
