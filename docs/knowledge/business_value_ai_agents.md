# Бизнес-ценность и границы автономных агентов (Lilian Weng & AI Engineering)

Этот документ определяет стандарты проектирования агентов, которые приносят измеримую бизнес-ценность (ROI), и задает границы их автономности.

## 🏛️ Архитектура агента (LLM-powered)

Агент — это не просто промпт, это система, состоящая из:
1.  **Brain (LLM)**: Ядро управления и принятия решений.
2.  **Planning**: Разложение сложных бизнес-целей на подзадачи (Subgoal Decomposition).
3.  **Memory**:
    - **Short-term**: Контекст текущей сессии (In-context learning).
    - **Long-term**: Внешнее хранилище знаний (RAG, Vector DB), позволяющее агенту помнить историю клиента годами.
4.  **Tool Use**: Способность вызывать внешние API (CRM, Биллинг, Телефония).

## 📊 ROI vs Хайп: Транзакционный подход

Для того чтобы агент был полезен бизнесу, он должен быть **транзакционным**, а не "болтливым":
- **Цель**: Не "поговорить", а выполнить действие (забронировать встречу, квалифицировать лид, создать заказ).
- **Фильтрация юзкейсов**: Автоматизировать нужно только те задачи, где:
    - Снижаются операционные косты.
    - Исключаются ошибки человеческого фактора в рутине.
    - Есть четкий критерий успеха (Success Metric).

## 🛡️ Границы автономии

Проектировщик должен четко разделять:
- **Автономные зоны**: Поиск по базе знаний, первичная квалификация, сбор данных.
- **Зоны контроля (Human-in-the-Loop)**: Списание денег, изменение критических статусов, финальное подтверждение крупных сделок.

> [!IMPORTANT]
> Агент без системы планирования и внешней памяти — это просто "генератор текста". Для бизнеса это риск. Агент с HLV-гейтами и спецификациями — это надежный сотрудник.
