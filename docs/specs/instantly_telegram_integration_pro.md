# 🛠 Технический Фундамент: Instantly + n8n + Telegram (HITL)

Этот документ описывает архитектуру контроля AI-агентства по модели **Human-in-the-Loop (HITL)**.

## 1. Конфигурация Instantly.ai API V2
- **Базовый URL**: `https://api.instantly.ai/v2/`
- **Аутентификация**: `Authorization: Bearer <V2_API_KEY>`
- **Ключевые сущности**:
    - `Leads`: Добавление (`POST /leads`), удаление, перемещение по спискам.
    - `Campaigns`: Запуск/остановка, получение статистики.
    - `Unibox`: Работа с ответами клиентов (`GET /emails`).

## 2. Схема Webhooks (Real-time уведомления)
Для мгновенной реакции на ответы клиентов настраиваются вебхуки в Instantly:
- **Событие**: `Reply Received`.
- **Target URL**: Webhook-узел в n8n под управлением нашего агента.
- **Payload**: Содержит текст ответа, `email` лида и `campaign_id`.

## 3. Паттерн Telegram-Control (Админка)
Для обеспечения 100% контроля владельца агентства над процессом используется следующий воркфлоу в n8n:

### Шаг A: Узел Одобрения (Approval Gateway)
Когда ИИ подготовил "Lead DNA" или проект письма:
1. n8n отправляет сообщение в Telegram с Inline-кнопками:
   - `[✅ Одобрить и отправить]`
   - `[✏️ Редактировать]`
   - `[🗑 Пропустить]`
2. Воркфлоу переходит в режим ожидания (**Wait for Webhook node**).

### Шаг Б: Безопасность
- **Whitelist**: Проверка `chat_id` отправителя. Только владелец (Вы) может нажимать кнопки.

## 4. Схема интеграции с LightRAG
Telegram бот может выступать интерфейсом к базе знаний:
- **Команда `/ask <вопрос>`**: n8n пересылает вопрос в API LightRAG и возвращает ответ из базы знаний проекта.
- **Команда `/status`**: Выводит текущее количество лидов в обработке и статистику кампаний.
