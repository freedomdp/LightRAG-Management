# SDD Pillar 6: VibeContract Formal QA Framework and Guardrail Logic

### 6. VibeContract: Формальное QA в вайбкодинге (arXiv)

**Концепция**: Решение проблемы "черного ящика" ИИ-генерации через создание формальной оболочки (VibeContract), которая окружает каждый логический блок кода.

**Структура VibeContract**:
1.  **Pre-conditions (Пред-условия)**: Какие данные и в каком состоянии *обязаны* прийти на вход (типы, диапазоны, не-пустота).
2.  **Post-conditions (Пост-условия)**: Какие данные и в каком состоянии *гарантированно* возвращаются (формат ответа, корректность структуры).
3.  **Invariants (Инварианты)**: Глобальные правила, которые не должны нарушаться внутри блока (например, "баланс счета не может быть отрицательным").
4.  **Exceptions (Исключения)**: Описание всех возможных ошибочных ситуаций, которые код *обязан* обработать.

**Механика "QA Gate"**:
- Перед кодингом Агент пишет VibeContract в комментариях или отдельном .yaml/.json файле.
- Инструмент валидации (hlv) прогоняет тесты на соответствие кода этому контракту *сразу* после генерации.
- Если контракт нарушен — код не принимается, Агент возвращается на этап исправления Спецификации.

**Связь с HLV**: Это и есть основа "Gate-driven" разработки. Никакого "задача выполнена" без прохождения формальных условий контракта.
