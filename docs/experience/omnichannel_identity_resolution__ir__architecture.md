# Omnichannel Identity Resolution (IR) Architecture

# Архитектура Identity Resolution и Омниканальности (Expert-Level)

Стандарт определяет алгоритмы сквозной идентификации и логику управления графом идентичности.

## 🔗 Логика склейки (Matching Logic)
### 1. Детерминированное сопоставление (Deterministic Matching)
- Принцип: Использование уникальных проверенных ID.
- Идентификаторы: Email, User ID, Номер телефона, налоговые ID.
- Точность: 100%. Любое несовпадение ведет к разделению профиля или вызову гейта.

### 2. Вероятностное сопоставление (Probabilistic Matching)
- Принцип: Статистическое моделирование вероятности.
- Параметры: IP-адрес, тип устройства, версия браузера, геолокация.
- Алгоритм:Similarity Score. Если score > Threshold (0.95), временное объединение до детерминированного подтверждения.

## 👤 Граф идентичности (Identity Graph)
Центральный реестр связок:
- Profile IDs: Мастер-ID пользователя.
- Anonymous IDs: Временные куки сессии.
- External IDs: IDFA (iOS), Google Advertising ID, FB Scoped ID.

## ⚙️ Алгоритм разрешения (Real-time Resolution Flow)
1. COLLECT: Сбор ID из веб-событий, бэкенда, CRM.
2. MATCH: Сравнение ID с графом через детерминированные и вероятностные фильтры.
3. MERGE: Объединение анонимных событий (Discovery) с авторизованным профилем (Conversion) в реальном времени.
4. ACTIVATE: Передача профиля в маркетинговый стек и AI-агенту.

## 🛡️ Ограничения и комплаенс (PII/GDPR)
- Data Clean Rooms: Работа без прямого доступа к PII.
- Consent Tracking: Каждый ID связан с Consent Token. Отозвал согласие -> блокировка персонализации во всех каналах.
