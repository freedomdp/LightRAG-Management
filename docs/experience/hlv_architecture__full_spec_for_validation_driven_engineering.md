# HLV Architecture: Full Spec for Validation-Driven Engineering

# Архитектура HLV (Human-LLM-Validation): Глубокий стандарт разработки

HLV — это архитектурная парадигма «Контрактного программирования для ИИ», предназначенная для устранения недетерминированности LLM.

## 1. Трехслойная модель
1. **Слой Человека (Human Layer)**: Статичный источник истины в `human/`. Содержит `glossary.yaml` (типы данных через JSON Schema), `contracts/*.yaml` (спецификации операций), `map.yaml` (индекс проекта).
2. **Слой LLM (LLM Layer)**: Динамическая реализация в `llm/`. LLM генерирует код, строго следуя контрактам и размечая логику маркерами.
3. **Слой Валидации (Validation Layer)**: Rust-валидатор (`hlv check`), проверяющий код на наличие маркеров и выполнение условий контрактов.

## 2. Система Маркеров (Traceability)
- **Синтаксис**: `// @hlv <ID>` или `# @hlv <ID>`.
- **Логика**: Каждое условие ошибки (`errors`) или инвариант (`invariants`) из контракта ДОЛЖНЫ иметь соответствующий маркер в коде реализации.
- **Цель**: Обеспечение сквозной прослеживаемости требований в коде. Валидатор выдает ошибку, если маркер отсутствует.

## 3. Контракты и Гейты (Gates)
- **Контракт**: Включает `inputs_schema`, `outputs_schema`, список `errors` с описанием условий (`when`) и `nfr` (нефункциональные требования).
- **Гейты (`gates-policy.yaml`)**: Обязательные проверки (coverage, security, unit tests), которые должны завершиться успешно для перехода задачи в статус `Passed`.

## 4. Специфика для Vibe Coding
Архитектура HLV позволяет сохранять высокую скорость разработки («вайб»), не жертвуя при этом надежностью и безопасностью, так как все критические моменты контролируются внешним валидатором, а не только вниманием LLM.
