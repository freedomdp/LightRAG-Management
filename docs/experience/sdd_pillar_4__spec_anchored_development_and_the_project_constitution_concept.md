# SDD Pillar 4: Spec-Anchored Development and The Project Constitution Concept

### 4. Spec-Anchored Development (Martin Fowler)

**Концепция**: Разделение на "Spec-First" (одноразовая спека для старта) и "Spec-Anchored" (спека — это живая Конституция проекта, которая существует весь цикл жизни ПО).

**Принципы "Привязанной Спецификации"**:
1.  **Immutable Logic**: Код никогда не должен содержать логику, которой нет в спецификации. Если возникла новая идея — она вносится в спеку, и только потом в код.
2.  **Versioning**: Спецификация версионируется вместе с кодом. Каждый коммит должен соответствовать состоянию спецификации на этот момент.
3.  **Tooling Support**: Использование инструментов (как LightRAG или Cursor), которые принудительно подтягивают спеку в контекст перед любой операцией кодинга.

**Проблема Drift (Дрейфа)**:
- Без "якоря" (spec-anchor) ИИ со временем начинает уходить в сторону своих паттернов, игнорируя стандарты проекта.
- Решение: Каждая итерация начинается с перечитывания Спеки сверху вниз (Surgical Ingest).

**Связь с HLV**: Валидатор HLV проверяет не только работоспособность кода, но и его "соответствие духу проекта" (alignment), сверяясь с Конституцией.
