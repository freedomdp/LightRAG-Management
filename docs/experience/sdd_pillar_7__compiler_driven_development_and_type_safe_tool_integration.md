# SDD Pillar 7: Compiler-Driven Development and Type-Safe Tool Integration

### 7. Compiler-Driven Development (Agentica)

**Концепция**: Устранение галлюцинаций в инструментоориентированных (Tool-use) ИИ через автоматическую генерацию JSON-схем функций напрямую из исходного кода с помощью типов компилятора.

**Преимущества архитектуры**:
1.  **Type-Safe Guarantee**: Агенту выдаются не просто текстовые инструкции ("вызови функцию X"), а строгие схемы параметров, проверенные компилятором (например, через Typia или TypeScript Reflection).
2.  **No Redundancy (Без дублирования)**: Мы не описываем аргументы функции дважды (в коде и в системном промпте). Схема для ИИ всегда на 100% совпадает с реальностью кода.
3.  **Build-time Validation**: Если подпись функции меняется, ИИ-схема обновляется автоматически при билде. Это предотвращает "Breakage" (поломку) агента при рефакторинге.

**Механика "Штатной Инженерии"**:
- LLM воспринимает функции как набор "жестких типизированных ручек".
- Ошибки типизации (например, строка вместо числа) ловятся еще до вызова модели через валидаторы в рантайме.

**Связь с HLV**: Это Layer 2 (LLM Generation). Мы сужаем пространство поиска для ИИ до такой степени, что он физически не может передать неверный тип данных в гейт-валидатор (Layer 3).
