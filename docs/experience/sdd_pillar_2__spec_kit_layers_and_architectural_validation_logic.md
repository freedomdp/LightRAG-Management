# SDD Pillar 2: Spec Kit Layers and Architectural Validation Logic

### 2. Spec Kit: Архитектура слоев (Habr)

**Концепция**: Разделение архитектурных знаний на два независимых, но связанных слоя: Системный дизайн и Функциональная спецификация.

**Детальная структура слоев**:
1.  **System Design (Архитектурный слой)**: 
    - Описывает взаимодействие компонентов, API-контракты, выбор библиотек.
    - Отвечает на вопрос "В какой среде и по каким правилам живет фича?".
    - Изменения здесь требуют обязательного апдейта Конституции проекта.
2.  **Functional Specification (Бизнес-лой)**: 
    - Описывает поведение фичи, пользовательские сценарии, краевые случаи.
    - Отвечает на вопрос "Что именно делает этот код?".
    - Генерируется на основе System Design, но не наоборот.

**Правила Программной Валидации (Spec Kit)**:
- **Инварианты структуры**: Проверка, не нарушает ли новый код заданную структуру папок и именование.
- **Инварианты зависимостей**: Запрет на импорты из слоев, которые не прописаны в System Design.
- **Генерация Схем**: На основе System Design автоматически генерируются интерфейсы и типы для ИИ, что гарантирует соответствие коду.

**Связь с HLV**: Инструмент hlv должен использовать System Design как эталон для проверки линтеров и архитектурных тестов. Любое отклонение считается Drift (дрейфом).
