# SDD Pillar 5: From Code to Contract and Formal Bug Handling Philosophy

### 5. From Code to Contract: Инверсия отношений (arXiv)

**Концепция**: В эпоху ИИ-ассистентов традиционная связь "Человек пишет код" заменяется на "Человек управляет контрактами". Прямое редактирование кода (Manual Patching) становится антипаттерном.

**Принципы инвертированной разработки**:
1.  **Contract-First Modification**: Если в коде найден баг или нужна фича, Агент обязан сначала изменить Контракт (или Спецификацию).
2.  **Code Regeneration**: Код перегенерируется или правится Агентом на основе обновленного контракта. Изменение "по месту" без обновления спеки ЗАПРЕЩЕНО.
3.  **Formal Bug Handling**: Баг рассматривается не как ошибка в коде, а как "несовершенство спецификации" или "пропущенный инвариант". 

**Логика "Программного Отсечения"**:
- Мы не просим ИИ "быть внимательнее". Мы накладываем жесткие рамки на генерацию через контракты. 
- Чем жестче контракт (Types, Ranges, Exceptions), тем меньше пространство для "галлюцинаций дрейфа".

**Связь с HLV**: Инструмент hlv в этой парадигме — это Compiler-Like валидатор, который проверяет код на соответствие новому контракту.
